## Phase 01 - Path helpers + bootstrap

Goal: ship the central `scripts/artifact_paths.py` module that owns every per-asset path convention; collapse `new_character.py`'s artifact-tree creation to just the 7 per-asset top-level dirs.

### Tasks

- [x] Create `scripts/artifact_paths.py` with public helpers covering all path patterns:
  - `asset_dir(character_dir, asset) -> Path`
  - `prompt_for(character_dir, asset, key=None) -> Path` — Pattern A/B emit `<asset>/prompt.md`; Pattern C with `key` emits `<asset>/prompts/<key>.md`.
  - `source_for(character_dir, asset, key=None) -> Path` — Pattern A/B emit `<asset>/source.png`; Pattern C with `key` emits `<asset>/source_strips/<key>_source.png`.
  - `normalized_for(character_dir, asset, frame_name=None) -> Path` — Pattern A emits `<asset>/normalized.png`; Pattern B/C emit `<asset>/normalized/<frame_name>.png` (caller owns the per-asset frame-naming convention).
  - `frames_dir(character_dir, asset) -> Path` — raises for Pattern A.
  - `source_strips_dir(character_dir, asset) -> Path` — raises for non-Pattern-C.
  - `report_for(character_dir, asset, step) -> Path` — `<asset>/reports/<step>.json`. For Pattern C, callers prefix `step` with the key (e.g. `down_0_normalize`).
  - `debug_for(character_dir, asset, name) -> Path` — `<asset>/debug/<name>` (caller supplies leaf filename).
  - `preview_for(character_dir, asset, key=None) -> Path` — Pattern B/`damage_poses` emits `<asset>/previews/damage_poses.gif`; Pattern C emits `<asset>/previews/<key>.gif`. **Raises** for `character_reference` (no previews per A3) and for Pattern A.
  - `rejected_dir(character_dir, asset) -> Path`
  - `needs_regen_dir(character_dir, asset) -> Path`
  - `manifest_validation_report(character_dir) -> Path` — cross-cutting, `artifacts/manifest_validation.json`.
  - `render_all_prompts_report(character_dir) -> Path` — cross-cutting, `artifacts/render_all_prompts.json` (per B3).
  - `package_summary(character_dir) -> Path` — cross-cutting, `artifacts/package_summary.md`.
- [x] Use `manifest.MULTI_MOTION_ASSETS` and `manifest.SINGLE_FRAME_ASSETS` from existing constants to branch on pattern (Pattern A = SINGLE_FRAME_ASSETS; Pattern B = `ALL_ASSETS - PATTERN_A_ASSETS - PATTERN_C_ASSETS`; Pattern C = MULTI_MOTION_ASSETS).
- [x] Add `tests/test_artifact_paths.py` covering every helper × every applicable pattern. Pin the post-008 layout in tests so future drift gets caught.
- [x] Update `scripts/new_character.py` `ARTIFACT_SUBFOLDERS` (currently 22 hard-coded entries) to just the 7 per-asset top-level dirs (`character_reference`, `character_movement`, `damage_poses`, `sideview_battler`, `face`, `picture`, `enemy`). Drop the nested `source/`, `normalized/`, `frames/`, plus the obsolete top-level `rejected/`, `needs-regeneration/`, `debug/`, `reports/`, `prompts/` entries.
- [x] Update `tests/test_new_character.py::test_bootstrap_creates_minimal_layout` to assert the 7 per-asset dirs exist and the obsolete subdirs do NOT.
- [x] Run `pytest tests/test_artifact_paths.py tests/test_new_character.py` and confirm green.

### Implementation evidence

- `scripts/artifact_paths.py`: 14 public helpers (12 listed in the spec plus `source_strips_dir` and `frames_dir` exposed as their own helpers). Pattern membership constants `PATTERN_A_ASSETS`, `PATTERN_B_ASSETS`, `PATTERN_C_ASSETS`, `PREVIEW_ASSETS` derived from `manifest`'s existing constants. Every helper validates the asset/key/frame-name combination and raises `ValueError` for invalid combos (Pattern A with frame_name, Pattern C without key, character_reference with preview_for, etc.).
